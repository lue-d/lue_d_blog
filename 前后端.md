# 前后端关联 — 实施概览

## 项目启动

项目为 Next.js 16.2.9 + Supabase 博客，运行在 `localhost:3000`。

```bash
# 启动（需先加载 nvm）
source ~/.nvm/nvm.sh && npm run dev
```

**前端**: Next.js 16.2.9 (Turbopack)，端口 3000
**后端**: Supabase 云端（PostgreSQL + Auth + Storage）

---

## 架构

```
浏览器 ──→ 前端页面（Next.js 静态导出）
                │
                ├── 公开页面（书法/摄影/感悟）：构建时从 Supabase 拉数据，生成静态 HTML
                │
                └── 管理后台（/admin）：Client Component，浏览器直连 Supabase API
                          │
                          ├── 登录 → Supabase Auth（邮箱 + 密码）
                          ├── 内容 → Supabase Database（CRUD）
                          └── 图片 → Supabase Storage（上传 + 公开 URL）
```

---

## 分支策略

| 分支 | 内容 | 用途 |
|------|------|------|
| `main` | 原始 MDX 版本 | 本地文件驱动，不依赖 Supabase |
| `master` | Supabase 关联版本 | 数据库驱动 + 管理后台，GitHub Pages 自动部署 |

两个分支通过 `git branch -f` 对调实现，`main` 保留原始版，`master` 为当前主力。

## 前端页面改造

### 公开页面（SSG）

| 路由 | 数据源 | 渲染方式 |
|------|--------|---------|
| `/calligraphy` `/photography` `/reflections` | `getContentList()` | async Server Component |
| `/[type]/[slug]` | `getContentData()` | async Server Component + `generateStaticParams` |

### 管理后台（CSR）

| 路由 | 功能 |
|------|------|
| `/admin/` | 仪表盘，显示三类内容数量 |
| `/admin/login/` | 邮箱密码登录 |
| `/admin/[type]/` | 列表页（表格 + 删除 + 编辑入口） |
| `/admin/[type]/new/` | 新建表单 |
| `/admin/[type]/[id]/edit/` | 编辑表单 |

管理后台页面全部是 Client Component，数据通过浏览器直连 Supabase API，不经过服务端。

### 组件

| 组件 | 用途 |
|------|------|
| `ContentForm.tsx` | 通用内容表单（新建/编辑复用），含图片上传 |
| `AdminLink.tsx` | Header 中的管理入口链接 |
| `Header.tsx` | 全局导航 |

## 图片加载优化

### 问题

`next/image` 默认通过 Next.js 服务器代理所有远程图片：

```
浏览器 → Next.js → 下载 Supabase 原图 → 压缩/转格式 → 返回
```

摄影大图（5-15MB）每次加载都要等服务端下载完成，极慢。

### 解决

所有 7 处 `next/image` 添加 `unoptimized` 属性：

```
浏览器 → Supabase CDN → 直出
```

涉及文件：
- `app/calligraphy/page.tsx`、`[slug]/page.tsx`
- `app/photography/page.tsx`、`[slug]/page.tsx`
- `app/reflections/page.tsx`、`[slug]/page.tsx`
- `components/ContentForm.tsx`

`next.config.ts` 全局设置 `images.unoptimized: true`。

## 静态导出与 GitHub Pages 部署

### 配置

`next.config.ts`：
```ts
output: "export",
trailingSlash: true,   // 解决 admin.html 与 admin/ 目录冲突
images: { unoptimized: true }
```

### `generateStaticParams`

| 路由 | 策略 |
|------|------|
| `/[type]/[slug]` | 从 Supabase 拉取所有已发布 slug |
| `/admin/[type]` | 硬编码 3 个类型 |
| `/admin/[type]/new` | 硬编码 3 个类型 |
| `/admin/[type]/[id]/edit` | 返回占位值（运行时客户端渲染） |

### 部署流程

```
git push master → GitHub Actions
  ├── checkout + node setup
  ├── npm ci
  ├── npm run build（读取 GitHub Secrets 中的 Supabase 凭据）
  └── peaceiris/actions-gh-pages → 推送 out/ 到 gh-pages 分支
```

### GitHub Secrets 配置

| Secret | 用途 |
|--------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase 项目地址 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | 客户端匿名密钥 |
| `SUPABASE_SERVICE_ROLE_KEY` | 构建时读取全量数据的密钥 |

## 已解决问题

| 问题 | 解决方式 |
|------|---------|
| 图片加载慢 | `unoptimized` 绕过 Next.js 代理，直连 Supabase CDN |
| admin 404 | `trailingSlash: true` 生成 `admin/index.html`，避免与子目录冲突 |
| Client Component 无法导出 generateStaticParams | 拆分为 Server Component（page.tsx）+ Client Component（*-client.tsx） |
| TypeScript 类型不兼容 | `data as unknown as Record<string, unknown>[]` |
