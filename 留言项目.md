## 一、功能概述

为博客添加留言功能，访客可在书法、摄影、感悟详情页留言，管理员在后台审核后公开展示。

### 核心规则
- **审核方式**：先审后发（访客提交 → 默认不公开 → 管理员通过 → 前端展示）
- **留言层级**：单层留言（不支持楼中楼回复）
- **必填字段**：留言内容、昵称、电子邮箱

## 二、数据库设计

### 2.1 新建表 `comments`

```sql
-- supabase/migrations/002_comments.sql

CREATE TABLE comments (
  id          uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  post_type   text NOT NULL,          -- 'calligraphy' | 'photography' | 'reflections'
  post_slug   text NOT NULL,          -- 关联文章 slug
  post_title  text NOT NULL DEFAULT '',-- 冗余存储文章标题（方便后台展示，即使文章被删也能识别）
  author_name text NOT NULL,          -- 留言者昵称
  author_email text NOT NULL,         -- 留言者邮箱（仅后台可见）
  content     text NOT NULL,          -- 留言正文
  published   boolean DEFAULT false,  -- false = 待审核, true = 已通过
  created_at  timestamptz DEFAULT now()
);

-- 索引
CREATE INDEX idx_comments_post ON comments(post_type, post_slug);
CREATE INDEX idx_comments_published ON comments(published);
CREATE INDEX idx_comments_created_at ON comments(created_at DESC);

-- updated_at 触发器
CREATE TRIGGER update_comments_modtime
  BEFORE UPDATE ON comments
  FOR EACH ROW EXECUTE FUNCTION update_modified_column();
```

### 2.2 RLS 策略

```sql
ALTER TABLE comments ENABLE ROW LEVEL SECURITY;

-- 公开：只能看到已审核通过的留言
CREATE POLICY "公开可读已通过留言" ON comments
  FOR SELECT USING (published = true);

-- 公开：可提交留言（强制 published = false，防止绕过审核）
CREATE POLICY "公开可提交留言" ON comments
  FOR INSERT WITH CHECK (
    auth.role() = 'anon' AND published = false
  );

-- 管理员：全部操作
CREATE POLICY "管理员可管理留言" ON comments
  FOR ALL USING (auth.role() = 'authenticated')
  WITH CHECK (auth.role() = 'authenticated');
```

## 四、各模块详细设计

### 4.1 `lib/comments-client.ts` — 前端 API

```ts
// 获取某篇文章已通过的留言
getComments(postType: string, postSlug: string): Promise<Comment[]>

// 提交新留言（published 固定为 false）
createComment(data: {
  post_type, post_slug, post_title,
  author_name, author_email, content
}): Promise<{ success: boolean; error?: string }>

// 获取各类型统计数据
getCommentStats(): Promise<{
  totalCalligraphy: number;
  totalPhotography: number;
  totalComments: number;  // 已通过
  recentComments: Comment[];  // 最新 5 条已通过留言
}>
```

### 4.2 `lib/comments-admin.ts` — 后台 API

```ts
// 获取全部留言（含未审核），支持筛选
getAllComments(filter?: {
  post_type?: string;
  published?: boolean;
}): Promise<Comment[]>

// 审核通过/驳回
togglePublish(id: string, published: boolean): Promise<void>

// 删除留言
deleteComment(id: string): Promise<void>
```

### 4.3 `components/CommentSection.tsx` — 留言区

**展示部分：**
- 留言列表，每条显示：头像占位（首字）、昵称、时间、内容
- 空状态："暂无留言，来发表第一条留言吧"
- 已通过留言按时间倒序

**表单部分：**
- 三个字段：昵称、电子邮箱、留言内容
- 前端校验：必填、邮箱格式
- 提交按钮 + loading 状态
- 成功提示："留言已提交，审核通过后将公开展示"
- 失败提示：具体错误信息

**功能：**
- 列表：留言内容（截断前 80 字）、昵称、邮箱、关联文章、时间、审核状态

## 五、数据流

```
访客提交留言
    │
    ▼
comments-client.ts → Supabase anon key
    │
    ▼
数据库 INSERT (published = false, RLS 强制)
    │
    ▼
[管理员审核]
    │
    ▼
comments-admin.ts → Supabase service_role key
    │
    ▼
UPDATE published = true
    │
    ▼
前端可见（RLS: SELECT WHERE published = true）
```

## 六、边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 文章被删除 | 留言保留，`post_title` 冗余字段仍可识别，后台显示"(文章已删除)" |
| 空留言内容 | 前端校验，禁止提交 |
| 邮箱格式错误 | 前端正则校验 `/\S+@\S+\.\S+/` |
| XSS 攻击 | React 默认转义文本内容，不渲染 HTML |
| 重复提交 | 按钮提交后 disabled + loading，防止连点 |
| 留言内容过长 | 限制 2000 字，前端 + 数据库 CHECK 约束 |
| 首页性能 | 统计数用 `count` 查询（不取全量），最新留言只取 5 条 |
| 移动端 tooltip | hover 不生效，改用点击展开/收起 |
